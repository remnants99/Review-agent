# 任务：审阅指定论文，并生成结构化中文审稿笔记

## Agent 硬性接口规则

以下规则用于让批处理 agent 能够稳定调用本提示词。用户可以修改审稿标准和输出结构，但请尽量保留本节中的占位符和约束。

- 本次只审阅指定论文：`{{input_file}}`。
- 输入目录为：`{{input_dir}}`。
- 输出目录为：`{{output_dir}}`。
- 最终 Markdown 审稿结果必须写入：`{{output_file}}`。
- 不要审阅 `{{input_file}}` 以外的其他论文；如果同目录下存在与该论文同名或明显相关的 `.tex`、`.md`、`.bib`、附录或补充材料，可以作为辅助材料参考。
- 不要把多篇论文的内容混合在同一份审稿文件中。
- 不要只在最终回复中给出审稿内容；必须创建或更新 `{{output_file}}`。
- 如果无法读取论文、无法可靠完成审阅或无法写入输出文件，请在最终回复中明确说明失败原因，并尽量写入一份包含失败原因的 Markdown 到 `{{output_file}}`。
- 文件名由 agent 提供。请在 Markdown 文档开头写出你识别到的论文 title、作者和输入文件名。

请审阅输入文件 `{{input_file}}`。默认输入文件为 PDF；如果存在 `.tex`、`.md` 或补充材料，也一并参考。请生成一份 Markdown 审稿文件，输出到 `{{output_file}}`。

文件名规则：
- 批处理运行时请使用 agent 提供的 `{{output_file}}`；
- 论文 title 请写入 Markdown 正文标题或元信息中；
- 如需额外创建辅助文件，文件名应删除或替换文件系统非法字符，例如 `/ \ : * ? " < > |`。

输出语言以中文为主；必要时可以保留英文术语。

## Markdown 数学公式渲染规则

为保证 Markdown 预览器尽可能正确渲染公式，请严格遵守：

- 行内公式使用 `$...$`，例如 `$x_t\in\mathcal X$`。
- 独立公式使用 `$$...$$`，并让开始和结束的 `$$` 各自独占一行，例如：

$$
R_T(x)=\sum_{t=1}^T f_t(x)-\sum_{t=1}^T f_t(x_t)
$$

- 独立公式的 `$$` 行不要缩进，不要放在 Markdown 表格单元格或代码块里。
- 不要使用 `\(...\)` 或 `\[...\]` 作为公式分隔符。
- 不要把公式放进 ```latex、```tex 或其他 Markdown 代码块中；代码块会导致公式以 TeX 源码显示。
- 不要用单反引号包裹公式，例如不要写 `` `$x_t$` ``。
- 如果必须展示原始 TeX 源码，请明确标注“原始 TeX 源码”，否则一律输出为可渲染公式。

## 预处理要求

在正式总结前，请先完成以下预处理：

1. 识别论文标题、作者、摘要、正文结构。
2. 为论文文本建立引用定位方式：
   - 所有引用均使用格式：`page X, lines Y-Z`。
   - 如果 PDF 没有原生行号，请按照文本提取后的页面内可见文本行自行编号。
   - 请在 Markdown 文档开头说明行号是如何生成的。
3. 所有重要定义、假设、定理、引理、公式都必须附带：
   - 中文解释；
   - 原文摘录；
   - page 和 line number；
   - 如果公式来自原文，请尽量转写为 Markdown 可渲染的 LaTeX。

若 PDF 是扫描版、文本提取失败、公式无法可靠识别，请明确标注：
> 本处由于文本/公式提取不可靠，以下内容可能需要人工核对。

不要编造不存在的 page、line number 或原文。
